# Enforcement Skills

These skills are designed to enforce architectural patterns, code quality standards, and best practices across the codebase. They act as guardrails during development.

## Available Skills

| Skill                                                         | Layer          | Priority | Description                                                                                                                                                                                                                                          |
| ------------------------------------------------------------- | -------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [API Layer](api-layer/skill.md)                               | infrastructure | high     | Enforces dual-provider service architecture (HTTP/Firebase) with factory pattern. Validates service factories use `CONFIG.SERVICE_PROVIDER`, implement repository interfaces, and return `T \| Error` (never throw). Includes templates for both providers. |
| [Architecture](architecture/skill.md)                         | cross-cutting  | high     | Enforces Clean Architecture with 4 mandatory layers (domain → infrastructure → application → ui). Validates dependency flow, domain purity (no React/framework imports), repository interfaces, and file naming conventions across feature modules.  |
| [Code Quality](code-quality/skill.md)                         | cross-cutting  | high     | Enforces ESLint, Prettier, TypeScript config, naming conventions (PascalCase/camelCase/SCREAMING_SNAKE), import organization (path aliases vs relative), and file structure patterns. Validates Babel path alias sync and proper handler naming.      |
| [Error Handling](error-handling/skill.md)                     | cross-cutting  | high     | Enforces centralized error handling via `manageAxiosError`, `T \| Error` pattern in services, Error-to-throw conversion in mutations, and UI guard order (loading → error → empty → success). Includes Spanish error message standards.              |
| [Forms & Validation](forms-validation/skill.md)               | ui             | high     | Enforces Yup schemas with Spanish error messages (string format), react-hook-form integration, form-to-payload adapters, and field validation rules (max length, coercion, optional fields). Links schema types to FormData.          |
| [Navigation Patterns](navigation-patterns/skill.md)           | ui             | medium   | Enforces typed navigation with RootStackParamList, nested stack navigators per module, type-safe hooks (useNavigation{Module}), route registration in `routes/` folder, and custom navigation params.                                                |
| [Performance Optimization](performance-optimization/skill.md) | cross-cutting  | medium   | Enforces FlashList for lists, React.memo for list items, useCallback for handlers, layout animations, image optimization, bundle size monitoring, and lazy loading patterns. Validates list performance with keyExtractor and getItemType.            |
| [Scalability Patterns](scalability-patterns/skill.md)         | cross-cutting  | medium   | Enforces module isolation, barrel exports (`index.ts`), avoiding circular dependencies, shared code in `@modules/core`, feature flags, and horizontal module scaling. Validates no cross-module direct imports.                                       |
| [Security Hardening](security-hardening/skill.md)             | cross-cutting  | high     | Enforces MMKV encrypted storage, secure token management, API key protection (`.env`), SSL pinning readiness, biometric authentication patterns, and input sanitization. Validates sensitive data never in AsyncStorage.                              |
| [State Management](state-management/skill.md)                 | application    | high     | Enforces decision matrix: React Query for server state (with query key conventions, cache invalidation), Zustand+MMKV for global UI state (theme, user prefs). Validates mutations invalidate queries, detail queries use enabled guards.             |
| [Testing Strategy](testing-strategy/skill.md)                 | cross-cutting  | high     | Enforces Jest + React Native Testing Library patterns, component testing with theme providers, hook testing, integration tests, and coverage thresholds. Validates test structure (AAA pattern) and mock conventions.                                |
| [Theme & Styling](theme-styling/skill.md)                     | ui             | high     | Enforces 5-mode theme system (light/dark/primary/secondary/premium), design tokens (spacing/typography/colors/borderRadius), component style factories with mode parameter, and animation duration constants. Validates no hardcoded hex/pixels.       |
| [UI Components](ui-components/skill.md)                       | ui             | high     | Enforces 3-tier component system (core/form/layout), theme token usage via `useTheme()`, style factories in `theme/components/`, barrel exports, and consistent component APIs (variant/size/disabled props). Validates React.memo for list items.    |
| [Accessibility](accessibility/skill.md)                       | ui             | high     | Enforces accessibility standards: `accessibilityLabel`, `accessibilityRole`, `accessibilityHint`, `accessibilityState`, minimum touch targets (44x44 pts), color contrast ratios (WCAG 2.1 AA), screen reader support (VoiceOver/TalkBack), and live regions for dynamic content. |
| [Analytics & Monitoring](analytics-and-monitoring/skill.md)   | cross-cutting  | medium   | Enforces factory-based analytics providers, typed event catalog (`ANALYTICS_EVENTS`), automatic screen tracking via React Navigation, crash reporting with Crashlytics, performance traces, and user context management on auth events.                |
| [Image Handling](image-handling/skill.md)                     | cross-cutting  | medium   | Enforces image selection (camera/gallery) with permissions, compression via `IMAGE_CONFIG`, upload to Firebase Storage or HTTP with `T \| Error` pattern, FastImage caching, loading/error placeholders, and image validation before upload.           |
| [Offline-First](offline-first/skill.md)                       | cross-cutting  | high     | Enforces offline-first patterns using Zustand + MMKV for cache persistence, React Query `placeholderData` for offline fallback, mutation queue with retry, optimistic updates with dual rollback (React Query + Zustand), and connectivity detection — all with existing dependencies. |
