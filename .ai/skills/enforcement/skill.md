# Enforcement Skills

These skills are designed to enforce architectural patterns, code quality standards, and best practices across the codebase. They act as guardrails during development.

## Available Skills

- **[API Layer](api-layer/skill.md)**: Enforces dual-provider service architecture and error contracts.
- **[Architecture](architecture/skill.md)**: Enforces 4-layer Clean Architecture per module.
- **[Code Quality](code-quality/skill.md)**: Enforces naming conventions, imports, and formatting.
- **[Error Handling](error-handling/skill.md)**: Enforces centralized error handling and UI guard rails.
- **[Forms & Validation](forms-validation/skill.md)**: Enforces Zod schemas and react-hook-form usage.
- **[Navigation Patterns](navigation-patterns/skill.md)**: Enforces typed navigation and route definitions.
- **[Performance Optimization](performance-optimization/skill.md)**: Enforces performance best practices like FlashList and memoization.
- **[Scalability Patterns](scalability-patterns/skill.md)**: Enforces module isolation and scalable patterns.
- **[Security Hardening](security-hardening/skill.md)**: Enforces security best practices and secure storage.
- **[State Management](state-management/skill.md)**: Enforces state management decision matrix (Query vs Zustand).
- **[Testing Strategy](testing-strategy/skill.md)**: Enforces testing patterns and coverage requirements.
- **[Theme & Styling](theme-styling/skill.md)**: Enforces the 5-mode theme system and design tokens.
- **[UI Components](ui-components/skill.md)**: Enforces the 3-tier component system and style factories.
